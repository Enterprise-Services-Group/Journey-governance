# Journey-governance
An interactive governance assistant supporting the University of Melbourne’s Journey Management practice. Helps Journey Creators, Owners and Managers create, maintain, validate and govern journey assets consistently using TheyDo and evidence-based decision making.

This interactive web application provides role-based guidance, governance standards, decision support, health checks, and best-practice workflows to help teams create, maintain, validate, and steward journey assets consistently across the journey management ecosystem.

Features include:

- Journey Creator governance guidance
- Journey Owner maintenance and stewardship practices
- Journey Manager oversight and governance responsibilities
- TheyDo best-practice workflows
- Insight validation and lifecycle management
- Journey health assessments
- Governance checklists and decision guides
- University of Melbourne Journey Management principles

Built as a standalone HTML application with no backend dependencies, suitable for local use or deployment via GitHub Pages.
